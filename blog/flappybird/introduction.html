Making Flapus-Birdus in Ceramic

Introduction

In this series of tutorials, you'll learn how to accomplish a variety of things in Ceramic, a 2D media & game engine for Haxe.

Now, Haxe is a great language. It can transpile to a variety of languages for cross-platform builds, and it has an insane amount of options when choosing a game engine. It's even been used to make popular games such as Dead Cells, Papers Please, and Northguard! One of the newer avaliable engines is ceramic. Ceramic is a Cross-platform 2D framework written in Haxe that can export natively to desktop (windows, mac, linux), mobile (ios, android), web (js + webgl) and to unity projects. It's extensible, so you can add support for sprites, nape & arcade physics, and more. The engine also has a nice configuring process, and can be configured through a ceramic.yml file and a Project.hx file. You can read more about configuration on the ceramic website. Because of the ability to compile to unity projects, ceramic projects can also be compiled to any platform that unity supports.

This game will help you learn a variety of things for ceramic, and how the developing process typically works. These tutorials will help you with learning, in no particular order:

  • arcade physics with
    • collisions
    • gravity/acceleration
  • loading and using custom fonts
  • loading and using images
  • custom logic(classes)
  • changing scene
  • non-blocking timer
  • input with scan codes
  • sprites with spritesheet animations
Let's begin with downloading the ceramic cli tool and creating a project. As long as you have haxe and haxelib installed, you can install ceramic by running haxelib install ceramic. Then, run haxelib run ceramic setup to install ceramic on your computer. When on Linux, you may need elevated permissions to install ceramic globally. Now you may run ceramic help to check if the tool is installed. You can see that there a quite a lot of avaliable commands, but only a few will be used in these tutorials. To create a new project, run ceramic init --name flapus-birdus. You will see that a new directory will be created, called "flapus-birdus".
To run the project, run
cd flapus-birdus
ceramic clay run web --setup

These commands change the cwd to the newly created ceramic project, and then runs it in the web. Electron is enabled by default, so a window will be created with the project. the --setup flag will ensure that the web target is set up correctly before running the project.

And that's it, you have a ceramic project! In the next tutorial, we'll set up the main scene with a background and go over the structure of it, along with how to add assets to the project. You can go over the general structure of a ceramic project in this guide on the ceramic engine website.
